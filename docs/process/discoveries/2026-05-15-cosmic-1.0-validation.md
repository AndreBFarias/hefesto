# 2026-05-15 — Validação do Hefesto-Dualsense4Unix em Pop!_OS COSMIC 1.0

> Sprint 103: FEAT-COSMIC-NATIVE-VALIDATION-01.

## Ambiente

- Pop!_OS 24.04 LTS.
- Kernel 6.18.7-76061807-generic.
- COSMIC DE 1.0.0 (`cosmic-comp` rodando, `XDG_CURRENT_DESKTOP=COSMIC`).
- Wayland nativo (`XDG_SESSION_TYPE=wayland`, `WAYLAND_DISPLAY=wayland-1`).
- XWayland também ativo em paralelo (`DISPLAY=:1`).
- DualSense USB Sony 054c:0ce6 conectado (`/dev/hidraw0`).
- Pacote `hefesto-dualsense4unix` versão 3.0.0 (instalação fonte com `.venv` editable).

## Resultados — Window detection

| Cenário | Backend selecionado | Resultado real |
|---|---|---|
| `DISPLAY=:1` + `WAYLAND_DISPLAY=wayland-1` (XWayland presente) | `XlibBackend` | retorna `{"wm_class": "unknown", ...}` para janelas Wayland nativas; **funciona** para janelas XWayland (Steam, Proton) — quando essas estão ativas, XlibBackend resolve `wm_class` via `_NET_ACTIVE_WINDOW`. |
| `WAYLAND_DISPLAY=wayland-1` sem `DISPLAY` | `_WaylandCascadeBackend` (portal → wlrctl → None) | retorna `None` em todas as tentativas (vide abaixo). |

### Portal XDG (xdg-desktop-portal-cosmic)

Confirmação empírica de que o `xdg-desktop-portal-cosmic` em uso **NÃO implementa o método `org.freedesktop.portal.Window::GetActiveWindow`**.

Logs do `WaylandPortalBackend` em sessão real (5 tentativas consecutivas):

```
tentativa 1: result=None, failures=1, warned=False
tentativa 2: result=None, failures=2, warned=False
tentativa 3: result=None, failures=3, warned=True
tentativa 4: result=None, failures=3, warned=True
tentativa 5: result=None, failures=3, warned=True
```

Warning único emitido na 3ª falha (conforme `_UNSUPPORTED_THRESHOLD=3`):

```
[warning] wayland_portal_unsupported compositor=COSMIC failures=3
hint="Compositor Wayland nao implementa
'org.freedesktop.portal.Window::GetActiveWindow' (COSMIC 1.0+ com
xdg-desktop-portal-cosmic atualizado ou GNOME 46+ necessario).
Cascade tentara wlrctl em seguida; se ausente, autoswitch fica inativo."
```

### Wlrctl (wlr-foreign-toplevel-management-unstable-v1)

Confirmação empírica de que o `cosmic-comp 1.0` em uso **NÃO expõe a interface `wlr-foreign-toplevel-management-unstable-v1`**.

Comando direto:

```bash
$ wlrctl toplevel list --json --state activated
Foreign Toplevel Management interface not found!
$ echo $?
1
```

`WlrctlBackend.get_active_window_info()` retorna `None` graciosamente (subprocess returncode != 0 → log debug + None).

### Conclusão Window Detection em COSMIC 1.0

| Cenário | Status |
|---|---|
| XWayland presente + janela XWayland (Steam, Proton): | **OK** via XlibBackend |
| XWayland presente + janela Wayland nativa (Firefox-Wayland, Files): | **FALHA** (XlibBackend só vê janelas XWayland) — autoswitch cai em `fallback.json` |
| Wayland puro sem XWayland (forçado via `unset DISPLAY`): | **FALHA TOTAL** — portal , wlrctl  → cascade retorna None |

**Conclusão prática para o usuário COSMIC hoje (2026-05-15):**
- Mantenha `DISPLAY=:1` ativo (XWayland default em Pop!_OS 24.04 COSMIC).
- Use `--force-xwayland` ao instalar para garantir GUI sob XWayland.
- Para jogos via Steam/Proton (caso primário de uso do Hefesto), o XlibBackend resolve `wm_class` corretamente — autoswitch funciona.
- Para apps Wayland nativos (Firefox-Wayland, GNOME apps), autoswitch fica inativo até `xdg-desktop-portal-cosmic` implementar `GetActiveWindow` OU `cosmic-comp` expor `wlr-foreign-toplevel-management`. Workaround: trocar perfil manualmente via tray/CLI/combo PS+D-pad.

## Resultados — Tray icon

Tray icon NÃO aparece em `cosmic-panel` apesar de:
- `cosmic-applets 1.0.12~1777401307~24.04~c003924` instalado via apt.
- `AyatanaAppIndicator3` 0.5.93 disponível via `gir1.2-ayatanaappindicator3-0.1`.
- `Indicator.new()` chamado sem erro pelo Hefesto após `GLib.timeout_add(500, ...)`.

Probe D-Bus do `StatusNotifierWatcher`:

```bash
$ gdbus call --session --dest org.freedesktop.DBus --object-path /org/freedesktop/DBus \
    --method org.freedesktop.DBus.NameHasOwner org.kde.StatusNotifierWatcher
(false,)
```

Conclusão: o applet "Área de status" do `cosmic-applets` **não está habilitado por default no painel**. Precisa adicionar manualmente via "Configurações > Painel > Applets". A sprint `FEAT-COSMIC-TRAY-FALLBACK-01` agora detecta isso e emite uma notification D-Bus orientadora uma vez por execução do daemon/GUI.

Próxima vez que o usuário adicionar o applet ao painel:
- `gdbus` acima retorna `(true,)`.
- `statusnotifierwatcher_available()` retorna `True`.
- Tray icon do Hefesto aparece e funciona normalmente (sujeito ao bug
  `cosmic-applets#1009` que pode afetar atualizações de menu em runtime,
  workaround: recriar `Gtk.Menu` inteiro em vez de mutar items existentes).
